# Appendix A: Methods & Reproducibility

This appendix details the scoring protocol, data processing, and simulation workflow used in Chapter 2. All steps are designed for reproducibility and open testing.

### 1. Scoring Protocol – 35-Metric Compass

Scores are assigned using public historical consensus, guided by a structured prompt given to multiple LLMs (Grok, Claude, Gemini, etc.). The prompt enforces:

- Range: -10 to +10 per metric
    - Negative = left/collectivist/authoritarian tilt
    - Positive = right/market/libertarian tilt
- Rationale: 1–2 sentences only, evidence-based (no speculation)
- Output: Strict CSV format (Metric, Description, Score, Rationale) with quotes around any rationale containing commas

### 2. Compass Calculation

- Raw X = mean(A1–G3 scores) / 10 × 1.2
- Raw Y = mean(H1–M3 scores) / 10 × 3.0
- Rigidity penalty: variation_suppression = max(0, -mean(C2, I1–I3)/10) × uniformity_pressure = max(0, mean(H1–H3)/10)
Y_adjusted = Y × (1 + 1.5 × rigidity)
- X/Y clipped to [-1, 1]
- Zone assignment via thresholds (Prime Mutualism ≥0.8, Stressed Mutualism ≥0.3, Separatrix >–0.29, etc.)

### 3. 15×6 Grid Mapping

Metrics load rules via predefined mapping (see master grid doc):

- Rule 13 ← H1–H3, I1–I3, J3 (governance suppression metrics)
- Rule 10 ← E2, K1 (niche/military expansion)
- etc. (full mapping in [15×6 Master Grid reference](https://github.com/MementoMori15x6/15x6-sim/blob/main/docs/15x6_master_grid.md))

Grid percentages: normalized row deltas from mean adjusted score, scaled to 100%.

### 4. Drift & Shock Simulations

- Creep: gradual negative delta on target metrics (e.g., L1–L3 succession, G1–G3 cheater detection)
- Shocks: discrete deltas at historical events (e.g., Commodus 180 CE, barracks emperors 235 CE)
- Cannibalization: when parasitism >18%, subtract fraction of mutualism mean
- Kill-switch: export ratio ≤1.0–1.1 signals thermodynamic tipping point

All logic in `simulate.py` (repo root). Data CSVs in `/data/`.

### 5. Reproducibility Checklist

- Run `simulate.py` with any consensus CSV to regenerate X/Y, grid, drift, etc.
- Figures generated via matplotlib/seaborn in Colab (code in repo notebooks)
- Open to community testing/refinement — fork, tweak parameters, submit PRs.

This protocol is proposed, not final. All scores provisional and open to empirical stress-testing.
